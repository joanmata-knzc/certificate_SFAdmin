Automation

Automatización y Lógica de Negocio

Hacer que Salesforce trabaje solo: validar datos, enviar emails, actualizar registros y guiar usuarios.

📌 Herramientas por orden de preferencia en el examen:
1. Flow Builder → La herramienta estándar para todo. Es la respuesta preferida.
2. Validation Rules → Para calidad del dato (errores al guardar).
3. Approval Processes → Para flujos de aprobación humana.
⚠️ Workflow Rules y Process Builder están obsoletos (Legacy). No los recomendará el examen para escenarios nuevos.

1. Flow Builder — El Motor de Automatización

Dónde: Setup → Process Automation → Flows
Herramienta visual (sin código) para construir lógica de negocio compleja.

Tipos de Flujos

Tipo de Flow ¿Cuándo se ejecuta? ¿Tiene interfaz de usuario? Caso de uso típico
Record-Triggered Cuando un registro se crea, edita o borra. No Enviar email al crear una Oportunidad. Actualizar un campo automáticamente.
Screen Flow Un usuario lo lanza manualmente (botón, link, etc.). Sí (pantallas con preguntas) Asistente para crear un pedido paso a paso. Formularios guiados.
Scheduled-Triggered A una hora/frecuencia programada. No Enviar recordatorio diario de tareas pendientes. Cierre automático de casos viejos.
Autolaunched (No Trigger) Llamado desde otro proceso (otro flow, Apex, etc.). No Lógica reutilizable que se invoca desde otros flujos.
Platform Event-Triggered Cuando se publica un Platform Event. No Integraciones en tiempo real con sistemas externos.

Elementos Clave dentro de un Flow

📋 Elementos de Datos

  • Get Records: Consulta registros de Salesforce.
  • Create Records: Crea uno o varios registros.
  • Update Records: Actualiza campos de registros existentes.
  • Delete Records: Elimina registros.

🧠 Elementos de Lógica

  • Decision: Un "if/else". Según una condición, toma un camino u otro.
  • Loop: Repite una acción para cada elemento de una colección.
  • Assignment: Asigna un valor a una variable.
  • Subflow: Llama a otro flow (reutilización de lógica).
💡 Tip de Examen: En Record-Triggered Flows, las acciones pueden ejecutarse Before Save (antes de guardar, más rápido, sin DML adicional) o After Save (después de guardar, permite crear/actualizar otros registros). Siempre que puedas, usa "Before Save" para actualizar el propio registro.

2. Validation Rules — Calidad del Dato

Dónde: Object Manager → [Objeto] → Validation Rules

🚦 ¿Cómo funcionan?

Son fórmulas booleanas (verdadero/falso). Si la fórmula devuelve VERDADERO, bloquea el guardado y muestra el mensaje de error definido.

  • ISBLANK(Phone) → Error si el teléfono está vacío.
  • CloseDate < TODAY() → Error si la fecha de cierre es en el pasado.
  • NOT(ISPICKVAL(Stage, "Closed Won")) && Amount > 100000 → Lógica combinada.

Funciones comunes: ISBLANK(), ISNULL(), ISPICKVAL(), AND(), OR(), NOT(), LEN(), TODAY(), NOW().

3. Approval Processes — Aprobaciones Humanas

Dónde: Setup → Process Automation → Approval Processes

✅ ¿Para qué?

Cuando una acción necesita la aprobación de otra persona antes de ejecutarse.

  • Ejemplo clásico: Un descuento mayor al 20% debe ser aprobado por el Director de Ventas.
  • Pueden ser de uno o varios pasos (ej: primero aprueba el manager, luego el director).
  • Se puede configurar quién aprueba: el manager del propietario, un usuario específico, o la cola.

⚙️ Acciones Disponibles

  • Al enviar a aprobación: Ej: bloquear edición del registro.
  • Al aprobar: Ej: cambiar campo Stage a "Aprobado", enviar email.
  • Al rechazar: Ej: revertir cambios, notificar al solicitante.
  • Al retirar: Ej: desbloquear el registro.

4. Otras Herramientas de Automatización

📋 Assignment Rules

Asignan automáticamente Leads o Cases al usuario o cola correcta según criterios.

  • Solo puede estar activa una regla a la vez por objeto.
  • Dentro de esa regla, puedes tener múltiples "Rule Entries" que se evalúan en orden.
  • Dónde: Setup → Lead Assignment Rules / Case Assignment Rules.

⏰ Escalation Rules

Reasignan o notifican sobre casos que no han sido atendidos en un tiempo determinado.

  • El tiempo cuenta según las Business Hours configuradas.
  • Se puede escalar a un usuario, cola, o enviar un email de alerta.
  • Dónde: Setup → Case Escalation Rules.
