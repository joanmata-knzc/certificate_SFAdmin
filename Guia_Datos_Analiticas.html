Data & Analytics Management

Gestión de Datos y Analítica

Importar, limpiar y visualizar datos. El 17% más práctico del examen.

1. Herramientas de Importación de Datos

Característica Import Wizard Data Loader
Límite de registros Hasta 50.000 Hasta 5.000.000
Instalación Navegador web (sin instalación) App de escritorio (Windows/Mac)
Objetos disponibles Solo objetos estándar clave + custom Todos los objetos
Detección de duplicados Sí, automática No automática (manual)
Operaciones Insert, Update, Upsert Insert, Update, Upsert, Delete, Hard Delete, Export
Programación automática No Sí (vía CLI)
Perfil mínimo requerido Admin o con permiso de importación Depende de los permisos del objeto

🔄 Operaciones de Datos

  • Insert: Crea registros nuevos.
  • Update: Actualiza registros existentes (requiere ID de Salesforce).
  • Upsert: Inserta si no existe, actualiza si existe (usa External ID).
  • Delete: Borra registros (van a la Papelera, recuperables 15 días).
  • Hard Delete: Borra permanentemente, sin papelera (solo Data Loader).

🧹 Limpieza y Calidad de Datos

  • Duplicate Rules: Alertan o bloquean cuando se intenta crear un registro duplicado.
  • Matching Rules: La lógica que determina cuándo dos registros son "iguales" (ej: mismo email).
  • Data.com / Third-party tools: Para enriquecer datos de contactos y cuentas.
  • La limpieza proactiva de datos es siempre más barata que corregirla después.
💡 Tip de Examen: Si preguntan cuándo usar Import Wizard vs Data Loader, la clave está en el volumen (más de 50k → Data Loader) y en si necesitas Hard Delete o Export (siempre Data Loader).

2. Reports (Informes)

Dónde: Pestaña "Reports" en la barra de navegación superior.

Tipos de Reportes

Tipo Descripción ¿Permite gráficos? Caso de uso
Tabular Lista simple de registros sin agrupaciones. Como Excel. No Lista de todos los clientes, exportar datos.
Summary Agrupado por filas. Permite subtotales. Sí Ventas por comercial. Casos por agente.
Matrix Agrupado por filas Y columnas. Permite totales cruzados. Sí Ventas por comercial y por mes (tabla cruzada).
Joined Combina varios bloques de reportes diferentes en una sola vista. Sí (limitado) Comparar métricas de diferentes objetos en un solo reporte.

📋 Report Types

Define qué objetos y campos están disponibles para construir el reporte.

  • Salesforce tiene Report Types estándar (ej: "Accounts with Contacts").
  • Puedes crear Custom Report Types para combinaciones que no existen.
  • Clave: Si quieres un reporte de Cuentas que tengan Contactos, necesitas el report type "Accounts with Contacts". Si quieres Cuentas que pueden o no tener Contactos, necesitas otro report type diferente.

⚙️ Funcionalidades Clave

  • Filters: Filtra los registros que aparecen (ej: solo oportunidades cerradas este año).
  • Bucket Fields: Agrupa valores en categorías sin crear un campo nuevo (ej: "Pequeño/Mediano/Grande" según el Amount).
  • Cross Filters: Filtra por la existencia o ausencia de registros relacionados (ej: Cuentas SIN Contactos).
  • Row-Level Formulas: Fórmulas que se calculan para cada fila del reporte.

3. Dashboards (Paneles)

Dónde: Pestaña "Dashboards" en la barra de navegación superior.

📊 ¿Qué son?

Visualización gráfica de múltiples reportes en una sola pantalla. Cada componente del Dashboard se basa en un Report.

🎯 Componentes (Gráficos)

  • Bar / Column Chart: Comparar valores entre categorías.
  • Donut / Pie Chart: Proporción de partes sobre el total.
  • Funnel: Pipeline de ventas o servicio.
  • Metric / Gauge: Un número clave (ej: total de ventas del mes).
  • Table: Datos tabulares dentro del dashboard.

👁️ Running User — La Clave de Seguridad

  • El Dashboard muestra los datos según los permisos del "Running User", no del usuario que lo está viendo.
  • Si el Running User es el Admin, todos verán todos los datos.
  • Dynamic Dashboard: Cada usuario ve sus propios datos. El "Running User" es "el usuario que está viendo el dashboard". Limitado a 5 dashboards dinámicos por org en ediciones inferiores.
💡 Tip de Examen: Solo los reportes de tipo Summary y Matrix pueden ser fuente de un componente de Dashboard (Tabular y Joined no pueden, salvo que uses un Table component especial). Si preguntan sobre un Dashboard donde cada manager ve solo sus datos, la respuesta es Dynamic Dashboard.

4. List Views (Vistas de Lista)

📋 ¿Qué son?

Filtros guardados sobre un objeto que muestran una lista de registros que cumplen ciertos criterios. Más simples que los reportes pero más directas.

  • Ejemplo: "Mis Leads de esta semana", "Casos sin respuesta", "Oportunidades de Q4".
  • Se pueden compartir con todos los usuarios, con grupos específicos, o ser privadas.
  • Permiten edición inline (cambiar campos directamente desde la lista).
  • Pueden visualizarse como tabla o como Kanban (tarjetas agrupadas por Stage).
